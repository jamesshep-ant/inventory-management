Factory Inventory Management System

System Architecture, Tech Stack, and Data Flow

Overview

What it is

A full-stack inventory management demo with a Vue 3 SPA frontend talking to a Python FastAPI backend. All data is mock JSON loaded into memory at startup — no database.

Key idea

Four global filters (Time Period, Warehouse, Category, Order Status) live in a shared composable. Every view watches them and refetches. The backend filters in-memory lists and returns Pydantic-validated responses.

Data Flow

Example trace: user selects "January" in the Time Period filter while viewing Dashboard.

User interaction

FilterBar.vue <select> bound via v-model to selectedPeriod

Shared state mutation

useFilters.js singleton: selectedPeriod.value = '2025-01' — all components see it

Reactive watcher fires

Dashboard.vue: watch([selectedPeriod, ...], () => loadData())

Build filter payload

getCurrentFilters() → { warehouse: 'all', category: 'all', status: 'all', month: '2025-01' }

API client serializes request

api.js strips 'all' values → GET /api/orders?month=2025-01 (via Promise.all for parallel fetches)

FastAPI handler

main.py: apply_filters() + filter_by_month() run on in-memory lists; Pydantic validates response

In-memory filtering

List comprehensions skip None/'all'. Month match: '2025-01' in item['order_date']. Quarters map via QUARTER_MAP.

Response received

Raw data assigned to refs: allOrders.value = ordersData

Computed properties recalculate

statusData, orderHealthMetrics, topProducts, categoryData, revenueGoal all derive reactively

Vue re-render

SVG donut chart, tables (v-for with unique :key), and cards update automatically

State Management & Patterns

Composables (Singleton Pattern)

No Pinia or Vuex. Module-scoped refs act as singletons — importing the composable anywhere yields the same reactive instance.

  • useFilters4 filter refs, getCurrentFilters(), resetFilters()
  • useAuthMock user (locale-aware), always authenticated
  • useI18nen/ja, t(), currency switching, localStorage persistence

Architectural Patterns

  • Filter propagationFilterBar → useFilters refs → view watch() → reload
  • Derived dataRaw API data in ref, transformations in computed
  • API abstractionSingle api object; strips 'all' before serializing
  • Parallel fetchPromise.all for independent endpoints
  • Loading patternloading/error/data refs + try/catch/finally
